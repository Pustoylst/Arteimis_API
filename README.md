# Alpha-Soft Task Tracker (Django REST Framework)

REST API для управления проектами, задачами и комментариями внутри команды разработки.

## Почему именно такие сущности

При проектировании учитывались паттерны Jira, Trello, Asana, Яндекс Трекер, YouGile:

- Проект с участниками и владельцем: базовая зона видимости и разграничения доступа.
- Задача с автором и исполнителем: разделение ответственности за постановку и выполнение.
- Приоритет и статус: ядро планирования и контроля потока работ.
- Дедлайн: контроль сроков и фильтрация задач по времени.
- Комментарии к задаче: единая точка обсуждения, чтобы не терять контекст в чатах.

## Функциональность

- Регистрация сотрудников.
- Создание проектов и управление участниками (только владельцем проекта).
- Создание задач внутри проекта.
- Назначение исполнителя только из участников проекта.
- Фильтрация задач по:
  - проекту,
  - статусу,
  - приоритету,
  - исполнителю,
  - дедлайну (до даты/после даты).
- Пагинация списка задач.
- Комментарии к задачам.
- Автодокументация Swagger и ReDoc.

## Права доступа

- Владелец проекта:
  - управляет участниками,
  - редактирует и удаляет любые задачи проекта.
- Исполнитель задачи:
  - может менять только `status` и `priority` своей задачи.
- Автор задачи (если не владелец):
  - может менять только `description`,
  - может удалить свою задачу.
- Доступ к проектам, задачам и комментариям есть только у участников соответствующего проекта.

## Технологии

- Python 3.13
- Django 6
- Django REST Framework
- django-filter
- drf-spectacular (OpenAPI, Swagger, ReDoc)
- SQLite (по умолчанию)

## Локальный запуск

### 1. Клонирование и переход в папку проекта

```bash
git clone <repo_url>
cd alpha_soft_tracker
```

### 2. Создание и активация виртуального окружения

Windows (PowerShell):

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 4. Миграции

```bash
python manage.py migrate
```

### 5. Создание администратора

```bash
python manage.py createsuperuser
```

### 6. Запуск сервера

```bash
python manage.py runserver
```

## Документация API

- Swagger UI: `http://127.0.0.1:8000/api/docs/swagger/`
- ReDoc: `http://127.0.0.1:8000/api/docs/redoc/`
- OpenAPI schema: `http://127.0.0.1:8000/api/schema/`

## Основные endpoints

- `POST /api/register/` - регистрация пользователя.
- `GET/POST /api/projects/` - список и создание проектов.
- `POST /api/projects/{id}/add_participant/` - добавить участника.
- `POST /api/projects/{id}/remove_participant/` - удалить участника.
- `GET/POST /api/tasks/` - список и создание задач.
- `GET/POST /api/comments/` - список и создание комментариев.

## Пример фильтрации задач

```http
GET /api/tasks/?project=1&status=IN_PROGRESS&priority=HIGH&assignee=3&deadline_before=2026-05-01
```

## Качество кода

- Код соответствует PEP8 (структура модулей, имена сущностей, читаемость).
- Использованы сериализаторы DRF с валидацией бизнес-ограничений.
- Права доступа реализованы на уровне object permissions.
